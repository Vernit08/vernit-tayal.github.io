What is DataGenie? An AI-powered WhatsApp bot that lets anyone upload an Excel file and instantly get data analysis — without needing Excel, Power BI, or any technical tools. Just send a file and ask your question in plain English.

How does it work? When a user uploads an Excel file, the bot reads it using Pandas and generates a file summary. The user then types any question. Groq AI (LLama model) generates accurate Pandas code, Python executes it on the actual data, and the result is formatted and sent back. This two-step approach — AI generates code, Pandas runs it — ensures 100% accurate numbers instead of AI hallucinations.

Why WhatsApp? Most non-technical users — shop owners, sales managers, HR teams — already live on WhatsApp. They have Excel files but no idea how to analyse them. DataGenie brings data analysis to a platform they already use every day.

Cold Start Delay

Render free tier sleeps when inactive. First message after a long gap may take 30-50 seconds to respond.

✅ Fix: cron-job.org pings server every minute — reduces this significantly. Full fix requires Render paid plan ($7/month).

📨

Twilio Sandbox Limit

Free Twilio sandbox allows only 50 messages/day and requires users to join via a code before using the bot.

✅ Fix: Upgrade to Twilio production WhatsApp number (~$15/month) for unlimited messages with no join code required.

📁

Excel Files Only (.xlsx)

Currently only supports Excel files. CSV, JSON, Google Sheets not yet supported.

✅ Fix: CSV support takes ~2 hours to add. Google Sheets integration planned as next feature.

🧠

No Conversation Memory

The bot doesn't remember previous questions in the same session — each question is independent. New file upload resets the session.

✅ Fix: Conversation history can be added to the session dictionary — planned for next version.

📊

No Chart Visualisation

Bot provides text answers and Excel downloads but cannot send chart images directly on WhatsApp via free tier.

✅ Fix: Imgur API integration for chart hosting planned — would enable sending bar/line charts as images.

🔒

No Data Persistence

Uploaded files are stored temporarily in server memory. If Render restarts, session is lost and user must re-upload.

✅ Fix: Adding a database (PostgreSQL — free on Render) would persist sessions across restarts.
